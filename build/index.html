безопасность расчетов

Важная задача платформы развития бизнеса Payments – обеспечить безопасность расчетов. Для ее выполнения в рамках платформы реализовано несколько продуктов.

ЗАЩИТА ЭЛЕКТРОННЫХ СООБЩЕНИЙ (УФЭБС)

Обеспечение информационной безопасности при передаче электронных сообщений в Банк России. Для выполнения проверки кодов аутентификации и защитных кодов, шифрования и дешифрования электронных сообщений в программном продукте применяется СКАД «Сигнатура»

Возможности продукта:

  • Загрузка электронных сообщений и пакетов электронных сообщений из АБС клиента
  • Проверка контрольной суммы электронного сообщения, полученного из АБС клиента
  • Установка защитных кодов и кодов аутентификации на электронные сообщения и пакеты электронных сообщений, полученные из АБС клиента
  • Выгрузка и получение данных из Автоматизированного рабочего места клиента Банка России (новое) (АРМ КБР-Н)
  • Проверка подписи, поставленной в АРМ КБР-Н

ЗАЩИТА СООБЩЕНИЙ (SWIFT LAU)

Обеспечение передачи защитного ключа из АБС клиента в SWIFT-терминал в составе SWIFT-сообщения. Ключ формируется с использованием алгоритма хеширования HMAC-SHA256

Возможности продукта:

  • Подписание исходящих сообщений SWIFT с использованием алгоритма LAU перед их передачей в терминал SWIFT
  • Проверка подписи с использованием алгоритма LAU для входящих сообщений SWIFT, приходящих из терминала SWIFT

АРХИТЕКТУРНЫЕ ПРИНЦИПЫ

В разработке программных продуктов

  • Открытая, надежная, масштабируемая и событийно управляемая микросервисная архитектура
  • Разработка приложений (PBC) и сервисов ведется один раз для всех каналов одной командой
  • Компонуемая бизнес-архитектура на основе платформ развития бизнеса
  • Технологические платформы и каналы не содержат продуктовой логики
  • Стандартизированный технологический стек с ориентацией на ПО с открытым исходным кодом (open source) и на российские компоненты

Во заимодействии с клиентом

  • Омниканальная процессно-ориентированная среда взаимодействия с клиентом
  • Единая логика представления пользовательских сценариев (UX) во всех каналах
  • Компонуемые бизнес-решения на основе платформ развития бизнеса
  • Надежная и удобная система идентификации клиента в каждом канале
  • Роботизация и использование технологий искусственного интеллекта
